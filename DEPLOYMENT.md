## Pre-Deployment Checklist

### 1. Environment Variables
Create `.env.local` (already have `.env.example`):
```env
# Site URL (change to production domain)
NEXT_PUBLIC_SITE_URL=https://chery.co.id

# Contact (optional - for form submissions)
CONTACT_EMAIL=info@chery.co.id
WHATSAPP_NUMBER=6281234567890
```

### 2. Update Domain References
Files to update before going live:
- `src/app/layout.tsx` — `metadataBase` URL
- `src/app/contact/page.tsx` — phone/email/WhatsApp links
- `src/app/cibubur/page.tsx` — Google Maps links
- `src/app/makassar/page.tsx` — Google Maps links
- `src/app/pare-pare/page.tsx` — Google Maps links

### 3. Add Real Images
Place images in `/public/images/`:
```
public/images/
├── vehicles/
│   ├── omoda-5.jpg
│   ├── tiggo-5x.jpg
│   ├── tiggo-7-pro.jpg
│   ├── tiggo-8-pro.jpg
│   ├── tiggo-8-pro-hybrid.jpg
│   └── eq1.jpg
├── og-image.jpg (1200x630)
└── favicon.ico
```

### 4. Update Vehicle Data
Edit `src/lib/data/vehicles.ts`:
- Replace placeholder prices with real prices
- Update phone numbers, addresses, emails
- Add real vehicle specifications

## Deploy to Vercel (Recommended)

### Option A: Vercel CLI
```bash
# Install Vercel CLI
npm i -g vercel

# Deploy
vercel

# Production deploy
vercel --prod
```

### Option B: GitHub Integration
1. Push code to GitHub
2. Go to [vercel.com](https://vercel.com)
3. Import repository
4. Add environment variables
5. Deploy

### Option C: Manual Build
```bash
# Build
npm run build

# Output in /out folder (static export)
# Upload /out to any static hosting
```
